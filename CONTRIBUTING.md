# Contributing to IntDM

Thank you for your interest in contributing to IntDM! 🎉

## How to Contribute

### Reporting Bugs
1. Check if the issue already exists in [Issues](../../issues)
2. Create a new issue using the **Bug Report** template
3. Include steps to reproduce, expected behavior, and macOS version

### Suggesting Features
1. Check the [Roadmap](docs/ROADMAP.md) for planned features
2. Open a new issue using the **Feature Request** template
3. Describe the feature and why it would be useful

### Pull Requests
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes following our coding standards
4. Write/update tests as needed
5. Submit a PR with a clear description of changes

## Development Setup

### Prerequisites
- macOS 14 Sonoma+
- Xcode 15+
- Swift 5.9+

### Getting Started
```bash
git clone https://github.com/YOUR_USERNAME/international-download-manager.git
cd international-download-manager
open IntDM.xcodeproj
```

## Coding Standards

- Follow [Swift API Design Guidelines](https://www.swift.org/documentation/api-design-guidelines/)
- Use SwiftLint for consistent code style
- Write meaningful commit messages (conventional commits preferred)
- Add documentation comments for public APIs
